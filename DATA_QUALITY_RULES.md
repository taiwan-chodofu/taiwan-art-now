# データ品質ルール（非交渉）

> このプロジェクトでは展示情報の**正確性が最優先**。
> 以下のルールに違反するコミットは禁止。

---

## 原則

1. **中文ページが source of truth**。英語は補完。
2. **推測・翻訳は禁止**。公式サイトに記載がない情報は空欄にする。
3. **施設名は公式サイトのタイトル/og:site_name から取得**する。AIの知識で設定しない。
4. **展覧会タイトルは公式ページの原文をそのままコピー**する。句読点・ダッシュの種類も変えない。

---

## スクレイパー設計ルール

### title_zh（中国語タイトル）
- **必ず中文ページから取得する**。英語ページからの翻訳は絶対禁止。
- 中文ページが存在しない施設は `title_zh` を空にする（推測しない）。
- CJK文字（一-鿿）を含むテキストは `title_zh` に設定する。

### title_en（英語タイトル）
- 英語版公式ページ、または展覧会の公式英語名から取得する。
- 英語版ページが存在しない場合は空にする。
- **中国語タイトルを翻訳して title_en を作成してはいけない**。

### 施設名（name_zh / name_en）
- 公式サイトの `<title>` または `og:site_name` meta から取得する。
- Facebook公式ページのページ名も有効なソース。
- **Claudeやその他AIの知識で施設名を推測してはいけない**。

---

## 手動JSON更新ルール

1. **必ず公式ソースを確認**してからJSONに書く。
2. 確認に使ったURL/ソースをコミットメッセージに明記する。
3. `validate_data.py` を実行して0エラーであることを確認してからコミット。

---

## 検証コマンド

```bash
python validate_data.py           # エラーチェック
python validate_data.py --verbose # 警告も表示
```

---

## 正確性確認の優先順位

| ソース | 信頼度 | 用途 |
|--------|--------|------|
| 公式中文ページ | 最高 | title_zh, name_zh |
| 公式英文ページ | 高 | title_en, name_en |
| 公式Facebook/Instagram | 高 | 施設名、展示情報 |
| ARTouch記事 | 中 | 参考（記事タイトルは編集されている） |
| ART iT記事 | 中 | 英語名の参考 |
| Wikipedia | 中 | アーティスト英語名 |
| Claude/AIの知識 | **使用禁止** | 推測の温床 |
